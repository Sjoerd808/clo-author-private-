Repository Demo

clo-author: Skill Usage Demo

This page demonstrates how to use the skills provided by this repository, both inside a research workflow and as an external skill provider for another local repository.

1) Core skill commands

SkillUse caseExample
/discoverFind literature/data and shape project scope./discover --lit --data "minimum wage and youth employment"
/strategizeDesign identification strategy and pre-analysis plan./strategize --design did --outcome employment
/analyzeImplement analysis in R/Python/Julia./analyze --lang python --task event-study
/writeDraft paper sections from analysis outputs./write --section results
/reviewRun critical review (paper/code/peer)./review --peer qje
/submitPrepare final submission and replication package./submit --journal aej-applied

2) Generate HTML outputs

Produce project-level and component-level HTML reports.

# Project dashboard
make dashboard

# Detail reports
make report-peer-review FILES="quality_reports/reviews/peer_round1.md quality_reports/reviews/peer_round2.md quality_reports/reviews/peer_round3.md"
make report-code-audit FILE=quality_reports/reviews/code_audit.md
make report-strategy-review FILE=quality_reports/reviews/strategy_review.md
make report-quality-gate FILE=quality_reports/reviews/quality_gate_summary.md
make report-literature FILE=quality_reports/reviews/annotated_bibliography.md

3) External provider mode

Call this repository from another local project.

# One capability
make skill-partial TARGET_REPO=/absolute/path/to/consumer-repo CAPABILITY=dashboard

# One bundle
make skill-bundle TARGET_REPO=/absolute/path/to/consumer-repo BUNDLE=execution \
  CONTRACT=/absolute/path/to/consumer-repo/provider-contract.json

# Full pipeline
make skill-full TARGET_REPO=/absolute/path/to/consumer-repo \
  CONTRACT=/absolute/path/to/consumer-repo/provider-contract.json

4) End-to-end demonstration flow

  1. Run /discover to gather literature and candidate data.
  2. Run /strategize to lock in empirical design and assumptions.
  3. Run /analyze to execute scripts and produce tables/figures.
  4. Run /write to draft sections linked to generated evidence.
  5. Run /review for code/paper/peer stress-testing.
  6. Run make dashboard to produce a self-contained HTML project dashboard.

Outputs are written under quality_reports/ and can be opened directly via file://.

5) Validation before use

make validate

This confirms dashboard/report/provider scripts are callable and ready before a run.
